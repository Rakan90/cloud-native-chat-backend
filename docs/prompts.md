# Prompts (AI-assisted parts)

This appendix records the prompts used during development. Per the course requirements, every prompt that drove generated code or generated documentation in this repository is listed below.

## 1. Audit prompt (project status check)

Prompt sent to Claude (Opus 4.7) to audit what was already done in the repository before continuing the work:

> Act like a strict, evidence-based software project auditor and requirements tracker for a university DevOps/Cloud project. Scan EVERYTHING inside the project folder/repository I provide and produce a clear, non-opinionated status report: what is implemented (with proof), what is missing or incomplete (with proof), what I should do next (minimal, concrete steps). You must NOT add extra requirements. … (full text in the conversation transcript)

Output: a structured Markdown report identifying that the Terraform files were empty, the GitHub Actions workflow only built (didn't push or deploy), the 15-factor doc was empty, and committed `.env` files contained secrets.

## 2. Implementation prompt (build the missing pieces)

Follow-up:

> Let us do all things in the terminal together.

This drove all subsequent work — fixing the SIGTERM bug, removing committed `.env`s, writing the Terraform stack, the GitHub Actions deploy workflow, the IAM user/policy, and the bootstrap/destroy scripts.

## 3. Specific generation prompts (implicit, derived from the conversation)

The following artifacts were generated by the assistant from natural-language instructions in the same conversation thread; the system prompt and the audit report above are the effective inputs.

| Artifact | Generated content |
|---|---|
| `infrastructure/main.tf` | VPC + subnets + IGW + NAT + route tables; ALB + target groups + listener rule; RDS Postgres in private subnets; ECR repos with `force_delete`; ECS Fargate cluster; task definitions with Secrets Manager-mounted secrets; ECS services with `health_check_grace_period`; IAM execution + task roles; CloudWatch log groups. |
| `infrastructure/variables.tf` | Region, project name, VPC CIDR, DB credentials (password sensitive), JWT secret (sensitive), image URIs, desired counts. |
| `infrastructure/outputs.tf` | ALB DNS, ECR URLs, ECS cluster/service names, RDS endpoint, region. |
| `.github/workflows/deploy.yaml` | AWS auth, ECR login, parallel build/tag/push for both services, `aws ecs update-service --force-new-deployment` on both services, `aws ecs wait services-stable`. |
| `scripts/bootstrap.sh` | Single-command provision: `terraform apply`, ECR login, build/push images, force deployment, wait for services-stable. |
| `scripts/destroy.sh` | Empty ECR repos, `terraform destroy`. |
| `docs/15-factor-analysis.md` | Per-factor evidence pointing at concrete file paths in this repo. |
| `docs/architecture.md` | Component descriptions, request flow, deployment flow, trade-offs. |
| `user-service/src/db.js`, `message-service/src/db.js` | Added `ensureSchema()` for idempotent table creation; SSL toggle via `DB_SSL`. |
| `user-service/src/index.js`, `message-service/src/index.js` | Captured the `app.listen()` server reference and made the `SIGTERM` / `SIGINT` handlers actually call `server.close()`. |
